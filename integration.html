Simply Salt — Referral Scheduling API

Developer integration guide for the Peterman technician app  ·  v1  ·  July 2026
Live demo (real API, watch the calls): evanwalker55.github.io/simply-salt-partner-demo

What this is

Two HTTPS endpoints that let a technician offer a customer a water-softener salt delivery time on the spot and book it in one tap. No SDK, no OAuth, no data sync — plain JSON over HTTPS. Simply Salt's scheduling system computes the time live from its actual delivery routes; your app never sees the schedule, it just gets one proposed slot back.

1 · CheckTech confirms customer name, address, ZIP (pre-filled from the work order). App calls partner_check_availability → gets a date + 1-hour window in ~1s.
→
2 · AskTech reads the time to the customer: "They can be out Saturday between 10:45 and 11:45 — work for you?"
→
3 · BookOn yes, app calls partner_book_job with phone + water type. Customer and job are created in Simply Salt's system, tagged to your technician.
Design goal — zero typing: the technician is standing in a house that's already a job in your system. Pre-fill customer name, address, ZIP, and phone from the work order, and take the technician's name from their app session. The referral becomes two taps.

1POST /partner_check_availability

Read-only. Returns the proposed delivery slot: always the next upcoming Saturday, at the first open time from 9:00 AM with at least 30 minutes of clearance on each side of Simply Salt's existing stops. All ZIP codes are accepted.

Reference implementation

// Node/backend example — same two calls the live demo page makes (view its source for a browser version)
const BASE = "https://primary-6xmr-production.up.railway.app/webhook";
const KEY  = process.env.SIMPLY_SALT_PARTNER_KEY;   // production key, server-side only

async function call(path, body) {
  const r = await fetch(`${BASE}/${path}`, {
    method: "POST",
    headers: { "Content-Type": "application/json", "x-partner-key": KEY },
    body: JSON.stringify(body)
  });
  const data = await r.json();
  if (!r.ok) throw new Error(JSON.stringify(data));
  return data;
}

// step 1 — when the tech opens the referral screen
const slot = await call("partner_check_availability",
  { customer_name: job.customerName, address: job.street, zip: job.zip });

// step 2 — when the customer says yes
const booking = await call("partner_book_job", {
  customer_name: job.customerName, phone: job.phone, address: job.street, zip: job.zip,
  scheduled_date: slot.route_saturday, scheduled_time: slot.scheduled_time,
  water_type: "city", referred_by: session.technicianName, partner_company: "Peterman Brothers"
});

Rollout checklist

  1. Build against the demo key — it hits the real scheduler, so times returned are genuine. Tell us before load-testing.
  2. Add the referral tile/button to your existing referral screen; pre-fill fields from the active work order.
  3. Swap in the production key (delivered privately), stored server-side.
  4. Send one live test booking; Simply Salt confirms it landed with the referral note, then you ship.
Anything you want changed — extra fields, technician IDs instead of names, a callback webhook when the delivery completes (for commission reconciliation) — is a same-day change on our side. Just ask.
